# 🚀 Toolkit-29: Ecossistema de Engenharia de Elite

> **O Framework Definitivo para Scaffolding, Governança e Inteligência Artificial.**

O **Toolkit-29** não é apenas um gerador de projetos; é uma **fábrica de software autônoma** que injeta anos de expertise técnica e governança em cada linha de comando. Ele automatiza a criação de projetos enquanto garante conformidade absoluta com o **Protocolo 29**.

---

## 🏗️ 1. O que compõe o Toolkit-29?

O ecossistema é dividido em três pilares fundamentais que trabalham em harmonia:

### ⚡ Pilar A: O Motor (Factory)
Um motor CLI escrito em Node.js nativo (Zero-Dependencies) que realiza:
- **Onboarding Inteligente:** Configura caminhos de projetos e biblioteca de IA.
- **Scaffolding Dinâmico:** Gera projetos baseados em templates (Next.js, Vite, Node).
- **Injeção de DNA:** Copia fisicamente o Toolkit de IA e as Leis de Governança para o novo projeto.

### 🧠 Pilar B: O Cérebro (29-Toolkit)
Localizado em `/29-toolkit/.agent`, este é o repositório mestre de capacidades:
- **20 Agentes Especialistas:** Personas de IA configuradas para papéis como `orchestrator`, `security-auditor`, `frontend-specialist`, etc.
- **36 Habilidades (Skills):** Módulos de conhecimento profundo (ex: `nextjs-best-practices`, `supabase-hardening`, `clean-code`).
- **29 Modos Especializados:** Protocolos de auditoria avançada para Segurança, Arquitetura, Banco de Dados e Documentação.

### ⚖️ Pilar C: A Constituição (/rule)
Este é o diferencial competitivo do Toolkit-29. A pasta `/rule` contém as leis que governam o comportamento de qualquer IA ou humano no projeto:
- **CLAUDE.md:** O "Contrato Social". Define as leis de arquitetura (Siloed Architecture), integridade documental e proteção de segredos. É o que impede a IA de gerar código genérico ou inseguro.
- **GEMINI.md:** O "Manual de Execução". Orienta o Agente sobre como analisar contratos, planejar execuções e validar modificações usando o toolkit embarcado.
- **Por que é necessário?** Sem estas leis, os agentes de IA operam sob padrões genéricos. Com a pasta `/rule`, o projeto impõe um padrão de **Engenharia de Elite** automático.

## 🛠️ 2. Como Usar o Toolkit Globalmente

### **Setup Inicial**
Para habilitar os comandos mestre em qualquer terminal:

- **Mac / Linux:** `./setup-toolkit.sh` && `source ~/.zshrc` (ou `.bashrc`)
- **Windows (PowerShell):** `.\setup-toolkit.ps1`

### **Comandos de Elite**
- `29-init` : Cria um novo projeto do zero com toda a inteligência e governança embarcada.
- `29-toolkit` : Atalho para acessar/editar a central mestre de agentes e skills.

---

## 🧠 3. Gestão de Custom Skills

O Toolkit-29 foi desenhado para ser estendido. Você pode adicionar novas habilidades na pasta mestre:
`29-toolkit/.agent/skills/`

**Como criar uma nova Skill:**
1. Crie uma pasta `minha-skill/`.
2. Adicione um `SKILL.md` definindo os metadados e instruções.
3. Adicione pastas opcionais `scripts/` e `references/`.
4. Na próxima vez que você rodar `29-init`, essa nova skill será parte do toolkit copiado para o seu projeto.
