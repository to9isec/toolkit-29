# 🚀 Toolkit-29: Ecossistema de Engenharia de Projetos

> **É um Framework auxiliar para criação de estruturas de projetos, com foco em Governança, Arquitetura de Software e Inteligência Artificial.**

O **Toolkit-29** é uma ferramenta de **auxilio para criação de projetos autonomos** que injeta expertise técnica criando pastas e arquivos necessarios para execução inicial de um projeto, auxiliando na estrutura para da capacidade de interagir com IA e seguindo diretrizes de governança definidas na seguir no projeto. Ele automatiza a criação de projetos estruturando a raiz e as pastas do projeto, garantindo conformidade, evitando reesturutação futura com o seu projeto através do nosso **Protocolo 29**.

---

## 🏗️ 1. O que compõe o Toolkit-29?

O ecossistema é dividido em três pilares fundamentais:

### ⚡ Pilar A: O Motor (Factory)
Um motor CLI em Node.js (Zero-Dependencies) que realiza:
- **Onboarding Inteligente:** Configura diretórios mestre e biblioteca de IA automaticamente.
- **Scaffolding Dinâmico:** Gera projetos baseados em templates (Next.js, Vite, Node).
- **Injeção de DNA:** Copia fisicamente o Toolkit de IA e as Leis de Governança (`/rule`) para o novo projeto.

### 🧠 Pilar B: O Cérebro (29-Toolkit)
Localizado em `/29-toolkit/.agent`, é o repositório mestre de capacidades:
- **Agentes Especialistas:** 20 personas configuradas para papéis técnicos específicos.
- **Skills (Habilidades):** Módulos de conhecimento profundo para stacks modernas.
- **Modos de Auditoria:** Protocolos avançados para Segurança, Arquitetura e Código.

### ⚖️ Pilar C: A Constituição (/rule)
A pasta `/rule` contém as leis que governam o projeto. É o "Contrato Social" que impede a IA de gerar código genérico ou inseguro, garantindo a integridade da arquitetura.

## 🛠️ 2. Instalação e Configuração (Setup Rápido)

O Toolkit-29 deve ser instalado em uma pasta fixa (Sua "Fábrica"). A partir de lá, ele criará projetos em qualquer outro local do seu sistema.

### **Instalação Automática**
Abra o terminal na pasta do Toolkit e rode o comando conforme seu sistema:

- **Mac / Linux:** 
  ```bash
  sh setup-toolkit.sh
  ```
- **Windows (PowerShell):** 
  ```powershell
  .\setup-toolkit.ps1
  ```

### **O que o Instalador faz por você?**
1.  **Verifica Pré-requisitos:** Valida se o Node.js está instalado.
2.  **Cria Comandos Globais:** Registra `29-init` e `29-toolkit` no seu sistema.
3.  **Auto-Start:** Oferece para criar seu primeiro projeto imediatamente.
4.  **Portabilidade Total:** O Toolkit detecta automaticamente se foi movido para uma nova pasta e se auto-ajusta.

---

## 🚀 3. Comandos Disponíveis

Após a instalação, você pode usar estes comandos de **qualquer lugar**:

*   `29-init` : Inicia o motor para criar um novo projeto blindado.
*   `29-toolkit` : Atalho rápido para entrar na central mestre de agentes/skills.

---

## 🆘 4. Resolução de Problemas (Troubleshooting)

### **Mac / Linux**
*   **Erro de Permissão:** Se ao rodar `./setup-toolkit.sh` der "Permission Denied", use o comando `sh setup-toolkit.sh`.
*   **Comando não encontrado após setup:** Rode `source ~/.bashrc` (ou `~/.zshrc`) ou reinicie o terminal.

### **Windows**
*   **Erro de Scripts Desabilitados:** Se o PowerShell bloquear a execução, rode:
    `Set-ExecutionPolicy RemoteSigned -Scope CurrentUser`
*   **Node.js não encontrado:** Instale via `winget install OpenJS.NodeJS` ou pelo site oficial.

---

## 🎯 5. Filosofia de Uso
**Não mova o Toolkit-29 para dentro de seus projetos.** 
Mantenha o "Toolkit-29" em um local central. Quando precisar de um novo projeto, apenas digite `29-init`. O motor cuidará de "dar à luz" ao seu novo software com toda a inteligência necessária, mantendo o seu ambiente de trabalho limpo e profissional.
